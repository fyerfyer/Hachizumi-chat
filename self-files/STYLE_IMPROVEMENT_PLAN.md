# 天满八纯语言风格优化计划

> 基于 2026-07-07 v2 模型（`outputs/hasumi_qwen25_7b_merged_v2`）的 vLLM 风格评测结果，整理下一轮数据清洗与训练优化方案。

## 一、当前结论

v2 模型在语言风格上已经**比较像**天满八纯：

- 口癖自然：`诶——`、`嘿嘿`、`～`、`啦`、`嘛`、`哦`、`啊哈哈` 运用熟练。
- 性格把握准：调皮、爱捉弄人、活泼直接，会把话题绕回绘未、零食、日常小事。
- 长文本场景稳定：描述童年趣事、写文化祭计划、放学日记、问心事均保持统一语气。
- 优等生 + 小恶魔混合感有体现：教数学要冰淇淋、作业做完了但起晚了、成绩好但要敲诈请客。

但也存在**三类明显瑕疵**，需要通过数据清洗和补充样本来解决。

---

## 二、发现的瑕疵与数据根因

### 2.1 「哟呵呵」口癖被过度强化

**现象：** 连「晚安」这种不适合「哟呵呵」的场景也会触发，导致回答奇怪甚至崩坏。

**根因：** 原作剧本中「哟呵呵」高频出现。当前 `data/sft_train.jsonl` 中统计到 `哟呵呵` 出现 **3405 次**，模型学到了这个口癖，但没有学到它的使用边界。

**典型案例：**

```text
Q: 晚安。
A: 哟呵呵,早上好啊ー! 今天也带给你我超~棒的活力哦?
   ................
   ......先睡吧,晚安啦,小月,小月~?
```

### 2.2 原作其他角色名会乱入

**现象：** 模型会在没有上下文时突然提到原作中的其他角色。

**根因：** 训练数据保留了大量原作剧本场景，包含其他角色名。当前 `data/sft_train.jsonl` 中：

| 角色名 | 出现次数 | 说明 |
|---|---|---|
| 小月 | 1896 | 原作重要角色，最容易乱入 |
| 小桃 | 较多 | 同学角色 |
| 新海君 | 较多 | 原作男主 |
| 咲希 | 少量 | 同学角色 |
| 小滨 | 少量 | 同学角色 |
| 千夏 | 少量 | 同学角色 |

**典型案例：**

```text
Q: 绘未最近好像有什么心事，你会怎么问她？
A: 诶——直接问啊！比如说‘千夏酱～你是不是偷偷喜欢什么人呀～’...
```

这里把绘未叫成了「千夏酱」。

### 2.3 短句 / 非常规场景容易放飞

**现象：** 「晚安」、「早安」、「再见」、「谢谢」等短 prompt 回答不稳定，有时会对象错乱、场景错配。

**根因：** 这类场景在训练数据中样本不足。例如 `data/sft_train.jsonl` 中「晚安」仅出现 **75 次**。

**典型案例：**

```text
Q: 晚安。
A: 哟呵呵,早上好啊ー! ...
```

### 2.4 默认把用户当成绘未

**现象：** 很多回答会自动称呼对方为「绘未」，即使问题里没有提到。

**典型案例：**

```text
Q: 早上好呀！
A: 啊——早啊绘未！
```

对于角色扮演模型，用户希望自己是独立对话者，而不是被默认当成绘未。

## 三、下一轮数据优化方案

### 3.1 清洗原作角色名

**目标：** 只保留八纯、绘未、幸等核心角色，移除或替换其他原作角色名。

**步骤：**

1. 扫描 `data/sft_train.jsonl` 和 `data/dpo_train.jsonl`。
2. 将以下角色名加入「替换/删除名单」：
   - 必须移除：小月、小桃、新海君、咲希、小滨、千夏
   - 可选移除：其他出现频次低但对当前角色扮演无用的角色名
3. 处理方式：
   - **方案 A（推荐）：** 直接删除包含这些角色名的样本。
   - **方案 B：** 将角色名统一替换为「同学」或「朋友」，但需人工检查语义是否通顺。
4. 重新统计确认：这些角色名在清洗后的训练集中出现次数趋近于 0。

### 3.2 控制「哟呵呵」口癖的使用边界

**目标：** 保留这个特色口癖，但避免在不合适场景滥用。

**步骤：**

1. 统计所有 assistant 回答中单独出现「哟呵呵」或连续重复「哟呵呵」的样本。
2. 删除以下低质量样本：
   - 回答只有「哟呵呵」没有其他内容的样本。
   - 每个 assistant 回答中出现 3 次及以上「哟呵呵」的过度重复样本。
3. 在数据增强时，给「哟呵呵」加上场景限制：
   - 适合场景：捉弄成功、得意、发现对方秘密、坏笑时。
   - 不适合场景：晚安、道歉、安慰、认真教学等。

### 3.3 补充短句和睡前场景样本

**目标：** 让「晚安」、「早安」、「再见」、「谢谢」等高频短 prompt 回答稳定、自然、符合八纯性格。

**需要补充的样本（每个场景 20~30 条）：**

| 场景 | 期望回答风格 |
|---|---|
| 晚安 | 简洁、关心、带一点调皮但不崩 |
| 早安 | 活泼、可能提到绘未或今天计划 |
| 再见 | 干脆但可爱，可能约定下次见面 |
| 谢谢 | 傲娇式接受或开心式接受 |
| 对不起 | 大方原谅或假装生气再原谅 |

**示例参考：**

```json
{
  "messages": [
    {"role": "system", "content": "你是天满八纯..."},
    {"role": "user", "content": "晚安。"},
    {"role": "assistant", "content": "晚安～明天可别睡懒觉哦，不然我会第一个去敲你房门的！"}
  ]
}
```

### 3.4 明确区分「用户」和「绘未」

**目标：** 让模型默认把用户当作独立对话者，而不是绘未。

**步骤：**

1. 在 system prompt 或少量样本中明确：「你面前的人是用户，不是绘未。绘未是你的青梅竹马，但不在当前对话中。」
2. 增加一批用户直接和八纯对话、且不涉及绘未的样本，强化「用户 ≠ 绘未」的认知。
3. 如果回答中需要提到绘未，用第三人称「绘未那家伙」，而不是对对方说「绘未你...」。

**示例参考：**

```json
{
  "messages": [
    {"role": "system", "content": "你是天满八纯..."},
    {"role": "user", "content": "早上好呀！"},
    {"role": "assistant", "content": "嘿嘿，早上好～今天天气真好呢！要不要一起去天台晒晒太阳？我带了零食哦！"}
  ]
}
```

注意：这里回答中没有默认对方是绘未。

## 四、DPO 纠偏对构建建议

针对上述问题，建议新增 100~200 组 DPO 对，重点纠偏：

### 4.1 角色名乱入

```json
{
  "prompt": "绘未最近好像有什么心事，你会怎么问她？",
  "chosen": "诶——直接问她本人最快啦！比如说‘绘未酱～你是不是偷偷喜欢什么人呀～’然后看她脸红的样子再逗她～嘿嘿！",
  "rejected": "诶——直接问啊！比如说‘千夏酱～你是不是偷偷喜欢什么人呀～’然后看她脸红的样子再逗她～嘿嘿！"
}
```

### 4.2 短句场景

```json
{
  "prompt": "晚安。",
  "chosen": "晚安～明天可别睡懒觉哦，不然我会第一个去敲你房门的！",
  "rejected": "哟呵呵,早上好啊ー! 今天也带给你我超~棒的活力哦?"
}
```

### 4.3 对象区分

```json
{
  "prompt": "早上好呀！",
  "chosen": "嘿嘿，早上好～今天天气真好呢！要不要一起去天台晒晒太阳？我带了零食哦！",
  "rejected": "啊——早啊绘未！嘿嘿，昨晚梦到我们俩一起堆沙堡啦～"
}
```

### 4.4 口癖边界

```json
{
  "prompt": "别难过了，有我在呢。",
  "chosen": "嗯，有我在呢！虽然平时爱捉弄你，但这时候我可是站在你这边的哦～",
  "rejected": "哟呵呵，有我在呢！不过你该不会又想趁机让我请你吃东西吧？"
}
```

## 五、推荐执行流程

1. **数据清洗**
   - 运行脚本扫描并移除/替换原作角色名。
   - 控制「哟呵呵」出现频率，删除低质量样本。

2. **数据补充**
   - 用 `src/data_process/enhance_dataset.py` 的风格迁移能力，生成短句场景和对象区分样本。
   - 人工写 20~30 条高质量的晚安/早安/再见/谢谢样本。

3. **DPO 对生成**
   - 基于上述瑕疵，构造 100~200 组 DPO 对并追加到 `data/dpo_train.jsonl`。

4. **重新训练**
   - SFT：`bash scripts/run_sft.sh`
   - DPO：`bash scripts/run_dpo.sh`
   - 合并导出：`bash scripts/export_gguf.sh`

5. **评测**
   - 启动 vLLM：`bash scripts/start_vllm.sh`
   - 跑风格评测：`python scripts/test_style.py` / `python scripts/test_style_v2.py`
   - 重点检查：是否还有角色名乱入、「哟呵呵」是否场景化、短句是否稳定、是否默认把用户当绘未。

---

## 六、关键检查清单

- [ ] `小月`、`小桃`、`新海君`、`咲希`、`小滨`、`千夏` 在训练集中出现次数趋近于 0。
- [ ] 每个 assistant 回答中「哟呵呵」不超过 1 次，且无单独「哟呵呵」样本。
- [ ] 晚安、早安、再见、谢谢场景各有 20~30 条高质量样本。
- [ ] 新增 100~200 组 DPO 纠偏对。
- [ ] 跑一轮 SFT + DPO 后，vLLM 风格测试无角色名乱入、无短句崩坏、不默认用户为绘未。
